01 · Principles

What this system fixes

The current site mixes glassy lavender surfaces, italic serif headlines, and three or four different button styles. This system replaces all of that with a single, calm, consistent vocabulary.

01

White, always

Page backgrounds are pure white (#FFFFFF). No gradients, no tinted lavender washes, no glass. Color appears only on content.

02

One action button

Every primary action across every page uses the purple-gradient pill — same shape, same colors, same uppercase label, same arrow.

03

Sans-serif only

Inter for everything. No Cormorant, no Playfair, no Cinzel, no italic decorative cuts. Hierarchy comes from size and weight, not style.

04

Restrained surfaces

Containers are defined by 1px neutral borders. Shadows are barely-there. The lotus mark and the action button are the only visual flourishes.

02 · Brand Mark

Lotus

The lotus is the only brand symbol. Always rendered in solid ink, never colorized, never on a colored container.

Clear space & misuse

Don't

05 · The Action Button

One button, every page

Every primary action across the site uses this exact button — Sign In, Add to Bag, Checkout, Subscribe, Apply, Save Address, Place Order. No exceptions, no per-page variants.

Reference

The pill-shaped purple gradient button you provided sets the canonical pattern: pill radius, left-to-right purple gradient, white uppercase label with letter-spacing, optional trailing arrow.

10 · Do & Don't

Rules of the system

Do

Use white as the page background. Define containers with thin neutral borders.

Pure #FFFFFF page.
Don't

Don't apply lavender or rose gradient washes to the page or to large containers.

Tinted page background.
Do

Use the canonical gradient pill for every primary action.

Don't

Don't ship pink, dark-blue, square, or serif buttons.

Do

Set every text element in Inter. Use weight and size to build hierarchy.

Skin, lit from within.
Don't

Don't mix Cormorant, Playfair, Cinzel, or italic decorative cuts into the UI.

Skin, lit from within.
Do

Reserve the gradient for the action button only.

Don't

Don't paint the gradient onto cards, banners, or backgrounds.

11 · Tokens

Drop into globals.css

Replace the existing Cloud Luxe block with these tokens. Map your brand-* aliases here so existing components inherit the new system without per-file edits.

:root { /* Brand */ --brand-purple-1: #c087fc; --brand-purple-2: #9869f7; --brand-purple-darker: #6536d4; --brand-gradient: linear-gradient(90deg, var(--brand-purple-1), var(--brand-purple-2)); --brand-gradient-hover: linear-gradient(90deg, #b577fb, #7a4fe0); /* Surface */ --bg: #ffffff; --ink-50: #f9f9fb; --ink-100: #f3f3f5; --ink-200: #e5e5ea; --ink-300: #c8c8cf; --ink-400: #8a8a93; --ink-500: #5a5a64; --ink-700: #2a2a31; --ink-900: #111114; /* Type */ --font-sans: 'Inter', -apple-system, system-ui, sans-serif; /* Radius */ --r-sm: 6px; --r-md: 10px; --r-lg: 16px; --r-pill: 999px; /* Shadow */ --shadow-button: 0 6px 16px rgba(152, 105, 247, 0.35); --shadow-button-hover: 0 8px 22px rgba(152, 105, 247, 0.45); } /* Reset legacy */ body { background: #fff !important; font-family: var(--font-sans); } .font-serif, .font-playfair, .font-cinzel, .font-cormorant { font-family: var(--font-sans) !important; }
